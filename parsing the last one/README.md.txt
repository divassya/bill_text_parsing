Извлечение счетов из двоичных файлов биллинга

main.py - основной код. 

Для всей работы нужно запустить этот файл в командной строке.
Остальных файлов, как decompress.py, classiter.py и select-cases.py, не нужно запускать. 
Они импортируются в main.py как готовые библиотеки.

Перед запуском нужно изменить dir_name на путь, где есть папки 1)data, 2)data_done, 3)data_res, 4)data_zip 5)python_parsing 
для работы с файлами. Например, в "C:/Users/Assiya.Karatay/Desktop/digital-summer-kcell/". 
Если нет одной из перечисленных папок, лучше её создать. 
data - исходные файлы. 
data_done - деархивированные файлы без первых 9 байтов.
data_res - папка для результатов.
data_zip - пустая папка, который используется в модуле decompress.py. 
Рекомендуется ее не удалять, чтобы не менять расширения и уменьшить время открытия/закрытия файлов.
В python_parsing программные коды decompress.py, classiter.py, select-cases.py и main.py.

main.py:
-вызывает остальные модули и нужные библиотеки;
-читает исходные файлы побайтово;
-инициализирует используемые параметры;
-производит сортировку и сбор информации;
-записывает результаты в файлы.

Общее введение по параметрам:
1) info - массив всех прочитанных байтов из файла
2) tmp - текущий байт info
3) buff - 60 байтов до tmp 
4) nbuf - количество байтов в buff, максимум 60


Модули decompress.py, classiter.py, select-cases.py и  main.py должны быть в одном файле. 
Их компиляция независима от местоположения других данных для обработки.

decompress.py нужен для удаления первоначальных 9 байтов в исходных файлах и деархивации.

classiter.py нужен для создания структур parskey и row_det. 
Parskey - структура ключей, для этой задачи есть 16 примеров ключей. 
row_det - структура для особых ключей, которые находятся при определенных значениях некоторых параметров.

select-cases.py понадобится для выборки изменения в определенных случаях.
